## Description

## Summary
Add internationalization (i18n) support to the Control UI, starting with Simplified Chinese (zh-CN).

## Changes
- Add lightweight i18n module (`ui/src/i18n/`)
- Add English (en) and Simplified Chinese (zh-CN) locale files
- Update `navigation.ts` to use i18n for tab titles and descriptions
- Support browser language auto-detection
- Support localStorage persistence for user preference

## Features
- `t(key)` - Get translated string
- `tf(key, params)` - Get translated string with interpolation
- `setLocale(locale)` - Change locale programmatically
- `getLocale()` - Get current locale
- `getAvailableLocales()` - List all available locales

## Translation Coverage
This initial PR covers:
- Navigation group labels (Chat, Control, Agent, Settings)
- Tab titles and descriptions
- Common action buttons (Save, Cancel, Apply, etc.)
- Status messages
- Error messages

## Next Steps (future PRs)
- Add language selector in Settings UI
- Translate remaining view-specific strings
- Add more locales (zh-TW, ja, ko, etc.)

## Testing
1. Set browser language to Chinese
2. Open Control UI
3. Verify Chinese text appears

Or manually:
```js
import { setLocale } from './i18n';
setLocale('zh-CN');
```

No, this PR sets up the i18n framework that supports multiple languages. Currently it includes:

- **English (en)** - default
- **Simplified Chinese (zh-CN)** - new

Adding more languages is easy - just create a new file in `src/i18n/locales/` (e.g., `ja.ts` for Japanese, `es.ts` for Spanish) and add it to the locales map.

The framework auto-detects browser language and falls back to English if no translation is available. Happy to add more languages in follow-up PRs!
