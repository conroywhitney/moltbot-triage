## Description

Fixes #3504

## Summary
Adds inline audio playback support for TTS-generated audio in the WebChat (macOS/SwiftUI) interface.

## Changes
- **InlineAudioParser**: Detects `MEDIA:` prefixed paths pointing to audio files (.mp3, .opus, .m4a, .ogg, .oga, .wav, .aac, .flac)
- **InlineAudioPlayerView**: SwiftUI component with play/pause button, progress bar, and file name display
- **ChatMarkdownRenderer**: Integrated audio parsing to render inline players in assistant message bubbles
- **Tests**: Comprehensive unit tests for audio path parsing

## Screenshots
The audio player displays:
- Play/pause button (circular, accent colored)
- Audio file name
- Progress bar with elapsed/duration time
- Graceful error handling for missing files

## Testing
- Added unit tests in `InlineAudioParserTests.swift`
- Tested pattern matching for various audio formats and path structures
