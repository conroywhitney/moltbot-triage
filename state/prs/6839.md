## Description

## Summary

Fix [CWE-400] vulnerability in `extensions/voice-call/src/tunnel.ts`.

**Problem**: `runNgrokCommand()` and `isNgrokAvailable()` spawn ngrok CLI without any timeout. If ngrok binary hangs (network issues, auth stuck), Promise never resolves, causing voice-call functionality to hang indefinitely.

**Impact**: DoS - voice-call operations can hang forever

**Fix**: Add setTimeout + SIGTERM to both functions (same pattern as existing `startNgrokTunnel()`)

## Changes

- `extensions/voice-call/src/tunnel.ts`:
  - `runNgrokCommand()`: Add 30s timeout with SIGTERM
  - `isNgrokAvailable()`: Add 5s timeout with SIGTERM
  - Both include `clearTimeout()` on close/error events
