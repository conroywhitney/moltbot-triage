### @codearranger (2026-02-01)

Re P3 (empty `.env` values): this is consistent with how every other optional var in `docker-setup.sh` already works (e.g. `OPENCLAW_DOCKER_APT_PACKAGES`, `OPENCLAW_EXTRA_MOUNTS`, `OPENCLAW_HOME_VOLUME` all write empty values when unset). The `upsert_env` function treats all listed keys uniformly, and Docker Compose / shell both treat empty and absent the same way for these vars (`-n ""` is falsy). Changing this behavior for just our new var would be inconsistent, and changing it for all vars is a separate concern outside the scope of this PR.
