<h2>Greptile Overview</h2>

<h3>Greptile Summary</h3>

This PR updates the `soul-evil` hook configuration resolver to support the new `HookConfigSchema` shape by merging top-level config with values provided under `env`, and adds support for a flat env format for purge settings (`purge_at`, `purge_duration`). The bundled hook handler is also updated to pass the hook config through without an unnecessary cast.

The main integration point is `src/hooks/bundled/soul-evil/handler.ts`, which calls `resolveSoulEvilConfigFromHook` before applying `applySoulEvilOverride` to replace `SOUL.md` content during agent bootstrap. The bulk of the behavioral change lives in `src/hooks/soul-evil.ts`, where config parsing now attempts to accept both legacy direct keys (`file`, `chance`, `purge`) and new-schema env-based keys.

<h3>Confidence Score: 3/5</h3>

- Mostly safe to merge, but the new config merge logic has edge cases that can mis-handle overrides or throw on malformed config.
- The change is localized and aligns with the stated goal, but `entry.env` is spread without a type guard (can throw at runtime) and the precedence/merge semantics between top-level vs env keys are ambiguous and may not match the intended backward-compat behavior.
