## Description

## Summary

Adds the `--thread-id` option to the `openclaw message read` CLI command to enable reading thread replies on Slack and Telegram.

## Problem

When trying to read Slack thread replies using the message tool or CLI, the `threadId` parameter wasn't being passed through. The CLI `read` command was missing the `--thread-id` option entirely.

## Investigation

Traced through the codebase and found:

1. ‚úÖ `slack.actions.ts` correctly reads `threadId` from params and passes it to `handleSlackAction`
2. ‚úÖ `handleSlackAction` correctly passes `threadId` to `readSlackMessages`
3. ‚úÖ `readSlackMessages` correctly uses `conversations.replies` when `threadId` is set
4. ‚úÖ The message tool schema includes `threadId` (from `buildSendSchema`)
5. ‚úÖ Existing test `slack.actions.test.ts` confirms threadId forwarding works
6. ‚ùå CLI `read` command was missing `--thread-id` option

## Changes

- Added `--thread-id <id>` option to the `message read` CLI command

## Testing

- [x] Existing test `slack.actions.test.ts` verifies the action handler works correctly
