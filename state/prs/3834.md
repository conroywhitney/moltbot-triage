## Summary
- Updates documentation to recommend `npm install -g moltbot@beta` instead of `moltbot@latest`
- Adds warning notes explaining that `@latest` currently points to v0.1.0 which lacks the CLI binary
- Updates 6 documentation files with the fix

## Problem
The npm `@latest` tag for `moltbot` points to v0.1.0, which:
1. Has no `bin` field in package.json
2. Doesn't expose the `moltbot` CLI command after global install
3. Causes the Windows installer to fail with ENOENT

This affects all platforms, not just Windows, but Windows users are particularly impacted since they can't easily work around it.

## Solution
Update documentation to recommend `@beta` until `@latest` is updated:
```bash
npm install -g moltbot@beta
# instead of
npm install -g moltbot@latest
```
