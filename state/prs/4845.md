## Description

## Problem

The documentation links in `docs/index.md` use absolute paths (e.g., `/start/getting-started`) which work correctly on the Jekyll site (docs.openclaw.ai) but **break when viewing the file directly on GitHub** (blob view) — they resolve to `github.com/start/getting-started` instead of staying within the docs folder.

## Solution

Changed all internal documentation links to use relative paths (`./path`) which work correctly in both contexts:
- **GitHub blob view**: navigates within docs folder ✓
- **Jekyll/GitHub Pages**: resolves correctly with baseurl ✓

## Changes

- Converted 49 internal links from absolute (`/foo/bar`) to relative (`./foo/bar`) paths
- Includes both markdown links and HTML anchor hrefs in the header section

## Testing

Verified all linked files exist in the docs directory.
