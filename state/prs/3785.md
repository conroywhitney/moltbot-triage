## Description

## Summary
Add proper error handling and proxy support for Telegram file downloads.

## Changes
- Add `TelegramDownloadError` class with typed error codes (`fetch_failed`, `http_error`, `api_error`)
- Add `TelegramDownloadOptions` type with optional `fetchImpl` parameter for custom fetch implementation
- Wrap fetch calls in try-catch blocks for better error messages

## Motivation
Users in regions where Telegram API is blocked (e.g., China, Iran) cannot access `api.telegram.org` directly. This change:
1. Provides clearer error messages when network requests fail
2. Allows users to pass a custom fetch implementation (e.g., with proxy) via the `fetchImpl` option

## Testing
Tested on Windows with simulated network failures.
