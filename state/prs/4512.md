## Description

## Summary

Add Model Context Protocol (MCP) server integration for Cursor IDE, enabling:
1. **OpenClaw tools in Cursor** - Use OpenClaw's AI agent, messaging, and session management directly in Cursor's Composer
2. **Cursor's models in OpenClaw** - Use Cursor subscription models (Claude, GPT-4, Gemini) as OpenClaw providers

## Motivation

Cursor IDE is increasingly popular among developers, and MCP is becoming the standard protocol for AI tool integration. This integration allows:
- Developers to use OpenClaw's multi-channel messaging (WhatsApp, Telegram, Discord, Slack) directly from Cursor
- OpenClaw users to leverage their Cursor subscription for AI models without separate API keys
- Seamless workflow between coding in Cursor and communicating through OpenClaw channels

## Proposed Implementation

### 1. MCP Server (`openclaw mcp serve`)

Exposes OpenClaw functionality to Cursor via MCP protocol:

**Tools:**
- `openclaw_chat` - Chat with OpenClaw AI agent
- `openclaw_send_message` - Send messages through channels
- `openclaw_list_sessions` / `openclaw_get_session` / `openclaw_clear_session` - Session management
- `openclaw_execute_command` - Run OpenClaw commands
- `openclaw_get_status` - Gateway/channel status
- `openclaw_list_models` - Available models

**Resources:**
- `openclaw://status` - Gateway status
- `openclaw://models` - Model catalog
- `openclaw://sessions` - Active sessions
- `openclaw://config` - Configuration

**Prompts:**
- `code_review`, `explain_code`, `generate_tests`, `refactor_code`, `debug_help`, `send_notification`

### 2. Cursor Model Provider (`openclaw mcp setup-models`)

Uses Cursor's Copilot Proxy to expose subscription models:
- `cursor/claude-sonnet-4`
- `cursor/gpt-4o`
- `cursor/o1`
- `cursor/gemini-2.5-pro`
- etc.
