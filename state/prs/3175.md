<h2>Greptile Overview</h2>

<h3>Greptile Summary</h3>

This PR adjusts CLI config validation so `moltbot tui --url <URL>` can run in “client mode” even if local gateway config is invalid. It does this by passing raw argv into `ensureConfigReady` (from both the Commander preAction hook and the route-first path) and allowing the `tui` command to bypass config invalidation when a `--url` flag is present. CI workflow is also updated to skip submodule update steps when `.gitmodules` is absent.

<h3>Confidence Score: 4/5</h3>

- This PR is largely safe to merge, with one edge-case around how `--url` is detected for bypassing config validation.
- Changes are small and localized to config-guard gating and CI submodule checkout behavior. The main concern is that using raw argv string matching can bypass config validation even when the `--url` value is missing/empty, which could hide config issues or produce confusing downstream failures.
