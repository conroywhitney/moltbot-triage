## Description

## Summary

Currently, Discord messages sent via Clawdbot are limited to plain text + optional media attachments. For structured data like email summaries, task lists, or formatted reports, this results in either ugly markdown tables (rendered as codeblocks) or walls of plain text. Discord's native embeds provide much better visual hierarchy and readability.

## Proposed solution

Add `embeds` parameter to the Discord `sendMessage` action that accepts Discord embed JSON objects per the Discord API spec:

## Alternatives considered

- Plain text with Unicode formatting (bullets + bold) ‚Äî works but lacks visual polish
- Images with pre-formatted content ‚Äî heavy and not editable
- Splitting into multiple messages ‚Äî creates notification noise

## Additional context

Discord API fully supports embeds: https://discord.com/developers/docs/resources/channel#embed-limits

This would improve readability for email check summaries, task/to-do lists, system notifications, and data reports.
