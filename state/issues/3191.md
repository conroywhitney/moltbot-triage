## Description

## Summary
When calling the `POST /api/v1/datasets` API, passing a `language` parameter returns an error: `Extra inputs are not permitted`. Currently, the dataset language cannot be set via the API and only uses the default value. We need to support this parameter.

## Proposed solution
Add a `language` field (e.g., `Chinese`, `English`) to the request body of the `POST /api/v1/datasets` API, allowing the dataset language to be set programmatically. The API should accept and process this parameter without rejecting it as an extra input.

## Alternatives considered
No alternative solutions. Currently, language can only be set manually via the frontend UI, not via API for batch/automated configuration.

## Additional context
- API endpoint: `POST /api/v1/datasets`
- Example request body:
```json
{
  "name": "test_2",
  "language": "Chinese"
}
Error response:
{
  "code": 101,
  "message": "Field: <language> - Message: <Extra inputs are not permitted> - Value: <Chinese>"
}
