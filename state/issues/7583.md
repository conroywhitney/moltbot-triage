## Description

## Summary
Assistant messages in webchat display raw template variable `[[reply_to_current]]` instead of resolved content.

## Steps to reproduce
1. Open Control UI webchat
2. Send a message to start a conversation
3. Observe assistant's response

## Expected behavior
Template variables like `[[reply_to_current]]` should be substituted with actual content before rendering in the UI.

## Actual behavior
The literal string `[[reply_to_current]]` appears in the assistant's message bubble, e.g.:

## Environment
- OpenClaw version: 2026.1.30
- Gateway OS: macOS (darwin)
- Client: 390px viewport width (iPhone 14 Pro equivalent)
- Install method: brew
