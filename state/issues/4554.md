## Description

## Feature Request

Convert markdown tables in outbound Slack messages to native [Block Kit Table Blocks](https://docs.slack.dev/reference/block-kit/blocks/table-block/) instead of current text-based workarounds (bullets/code).

## Current Behavior

The `markdown.tables` config offers:
- `"bullets"` — converts tables to bullet lists
- `"code"` — wraps tables in code blocks
- `"off"` — sends raw markdown (broken rendering)

None of these produce clean tabular output in Slack.

## Proposed Behavior

Add a new option `"blocks"` (or make it the default for Slack) that:
1. Detects markdown tables in outbound messages
2. Converts them to Block Kit `type: "table"` blocks
3. Sends via `blocks` parameter in `chat.postMessage`

## Example

Input markdown:
```
| Name | Status |
|------|--------|
| foo  | ok     |
| bar  | error  |
```

Output Block Kit:
```json
{
  "type": "table",
  "rows": [
    [{"type": "raw_text", "text": "Name"}, {"type": "raw_text", "text": "Status"}],
    [{"type": "raw_text", "text": "foo"}, {"type": "raw_text", "text": "ok"}],
    [{"type": "raw_text", "text": "bar"}, {"type": "raw_text", "text": "error"}]
  ]
}
```
