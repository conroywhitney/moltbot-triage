## Description

## Summary

When using `/model` to override the session model in the TUI, the footer continues to display the config default model instead of the active session model.

## Expected Behavior

After running `/model google/gemini-2.5-flash-lite`, the TUI footer should display:
```
... | google/gemini-2.5-flash-lite | ...
```

## Actual Behavior

The TUI footer continues to show:
```
... | anthropic/claude-sonnet-4-5 | ...
```

## Verification

The `session_status` tool confirms the override is working:
```
Model: google/gemini-2.5-flash-lite
```

But the TUI footer still shows the config default (`anthropic/claude-sonnet-4-5`).

## Steps to Reproduce

1. Configure primary model: `anthropic/claude-sonnet-4-5`
2. Start TUI: `openclaw tui`
3. Run: `/model google/gemini-2.5-flash-lite`
4. Observe footer - still shows Sonnet instead of Gemini
5. Verify with `session_status` tool - correctly shows Gemini

## Environment

- OpenClaw: 2026.1.30
- OS: Windows 10
- Config: Primary model with fallbacks configured

## Context

The TUI docs state the footer shows: "connection state + agent + session + **model** + think/verbose/reasoning + token counts"

The `/model` directive persists to the session when sent standalone, but the footer doesn't reflect this change.

The TUI footer also shows incorrect context information:

**TUI shows:** `125k/1.0m (12%)`

The footer is displaying:
- Model's theoretical max context (1M tokens for Sonnet)
- Percentage of model's max

But should display:
- Session's configured context budget (200k from config)
- Percentage of session budget

Same root cause: **TUI footer shows model defaults instead of session state**.
