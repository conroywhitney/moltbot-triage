## Description

## Summary

I deleted the gateway runner from a previous 1.29 install, and tried installing 1.30 and 2.1. Gateway file is missing.

## Steps to reproduce

0. Run the openclaw uninstall command, don't reboot
1. Delete the file that is running the gateway on boot
2. Try to install a newer version of openclaw via CMD
3. Complete the clawdbot onboard --install-daemon

## Expected behavior

The full installation capable of running the gateway, some repair functionality that restores missing core files.

## Actual behavior

The install says [*] existing openclaw installation detected, even when the uninstall command has run, and the folder .openclaw is empty. The installation isn't able to repair a damaged or missing gateway runner via doctor.

## Environment

- Clawdbot version: 1.30 to 2.1
- OS: Winows 10
