## Description

## Bug Description

When opening the Clawdbot dashboard with a `?session=` URL parameter, the dashboard ignores it and loads the previously-used session from localStorage instead.

## Steps to Reproduce

1. Have session A open in the dashboard (e.g., `agent:main:chiefofstaff`)
2. Close the dashboard
3. Open a URL with a different session parameter: `/chat?session=agent:main:project-6`
4. Dashboard loads session A instead of `project-6`

## Expected Behavior

The URL `session` parameter should take precedence over the cached `lastActiveSessionKey` in localStorage settings.

## Actual Behavior

The dashboard restores the session from `clawdbot.control.settings.v1` localStorage, ignoring the URL parameter.

## Context

This breaks integrations that link directly to specific sessions (e.g., a Kanban board with per-project sessions).

## Proposed Fix

In the control-ui initialization, ensure URL params are processed AFTER localStorage settings are loaded, and that URL `session` param overwrites the stored `sessionKey`/`lastActiveSessionKey`.

## Environment

- Clawdbot version: latest (npm)
- Browser: Chrome
- OS: macOS
