## Description

## Summary
When Moltbot’s token usage exceeds approximately 85% of the model’s context window, Clawdbot becomes completely unresponsive. The process cannot be recovered without rebooting the entire MacBook.

## Steps to reproduce
1. Start clawdbot on a MacBook Pro (M4).
2. Instruct Clawdbot to perform continuous work in Telegram.
3. Let the session continue until token usage reaches ~85%+ of the model’s context window
(model used: MiniMax M2.1).

## Expected behavior
What did you expect to happen?
Clawdbot should continue operating normally, or at least:
Gracefully handle near-full context windows, or
Fail safely (e.g. reset context, return an error, or recover automatically).

## Actual behavior
What actually happened?
Clawdbot becomes fully stuck / frozen.
No further responses or progress.
Restarting Clawdbot alone does not resolve the issue.
A full macOS reboot is required before Clawdbot can run again.

## Environment
- Clawdbot version:  2026.1.24-3   The same issue also exists in version 2026.1.27-beta.1.
- OS: Mac OS 15.6.1    Darwin 24.6.0
- Hardware: MacBook Pro (M4)
- Model: MiniMax M2.1
- Install method (pnpm/npx/docker/etc):  curl -fsSL https://molt.bot/install.sh | bash

**Additional reproduction case - compaction retry loop causes timeout**

Experiencing the same issue on macOS. The agent completes its work but then gets stuck in a compaction retry loop until it times out after 10 minutes.

**Environment:**
- Clawdbot version: 2026.1.24-3
- OS: macOS Darwin 24.3.0
- Model: github-copilot/claude-opus-4.5
- Install method: homebrew

**Logs showing the compaction hang:**

```
embedded run agent end: runId=dd153aeb-96f7-4fbb-b5c4-a4f3295a93c0
embedded run compaction start: runId=dd153aeb-96f7-4fbb-b5c4-a4f3295a93c0
embedded run prompt end: runId=dd153aeb-96f7-4fbb-b5c4-a4f3295a93c0 sessionId=473a06bf-... durationMs=245082
embedded run compaction retry: runId=dd153aeb-96f7-4fbb-b5c4-a4f3295a93c0
...
(no further progress)
...
embedded run timeout: runId=dd153aeb-96f7-4fbb-b5c4-a4f3295a93c0 sessionId=473a06bf-... timeoutMs=600000
```

**Also possibly related:** Before the compaction hang, the memory subsystem was stuck polling an OpenAI batch embedding job indefinitely (see #4370):

```
openai batch batch_697c1ab699408190925ac82a813ffc6f in_progress; waiting 2000ms
openai batch batch_697c1ab699408190925ac82a813ffc6f in_progress; waiting 2000ms
(repeats for minutes until batch finally shows "finalizing")
```

**Workaround:** Clearing the session files and restarting the gateway:
```bash
rm ~/.clawdbot/agents/main/sessions/*.jsonl
echo '{}' > ~/.clawdbot/agents/main/sessions/sessions.json
launchctl kickstart -k gui/$(id -u)/com.clawdbot.gateway
```
