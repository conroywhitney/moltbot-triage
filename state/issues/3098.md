## Description

### Summary
The docs mention "OAuth" for Anthropic subscription auth, but setup-tokens from `claude setup-token` require a different JSON format.

### The Problem
When configuring `auth-profiles.json` manually for Claude Max/Pro setup-tokens, using `type: "oauth"` with `accessToken` field fails silently 

### The Solution
Setup-tokens require:
```json
{
  "type": "token",
  "provider": "anthropic",
  "token": "sk-ant-[REDACTED]..."
}
```

**Not:**
```json
{
  "type": "oauth",
  "provider": "anthropic", 
  "accessToken": "sk-ant-[REDACTED]..."
}
```

### Suggested Fix
Add explicit example in the Anthropic provider docs showing the correct `type: "token"` format for setup-tokens generated via `claude setup-token`.

### Environment
- clawdbot 2026.1.24-3
- macOS
