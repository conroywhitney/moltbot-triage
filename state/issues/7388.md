## Description

## Description

After upgrading from 2026.1.30 to 2026.2.1, audio transcription via the legacy `audio.transcription.command` config path fails for larger audio files (>20KB / ~10+ seconds).

## Config

```json
{
  "audio": {
    "transcription": {
      "command": ["whisper", "{file}", "--model", "large-v3-turbo", "--output_format", "txt", "--output_dir", "/tmp"],
      "timeoutSeconds": 1800
    }
  }
}
```

## Observed Behavior

- **Short voice messages** (~5 sec, 16KB): Transcription works ✅
- **Longer voice messages** (~20 sec, 34-39KB): Transcription fails silently, raw audio bytes passed to agent instead of transcript ❌

## Manual Test

Running whisper manually on the same files works perfectly:
```bash
whisper ~/.openclaw/media/inbound/file_152---.ogg --model large-v3-turbo --output_format txt --output_dir /tmp
# Output: "Figure out what happened, fix it..." ✅
```

## Environment

- OpenClaw version: 2026.2.1 (worked in 2026.1.30)
- OS: macOS Darwin 25.2.0 (arm64)
- Node: v25.5.0
- Channel: Telegram
- Whisper: openai-whisper via Homebrew

## Notes

- This may be related to #5845 (`{file}` placeholder substitution bug in `tools.media.audio.models`)
- The legacy `audio.transcription.command` path was working as a workaround until this update
- No errors visible in gateway logs
