## Description

## Problem

When working with Clawdbot, I often want to run multiple parallel contexts:
- A long-running coding task in project A
- A quick question about groceries
- Monitoring a background build

Currently, spawned sessions run tasks and return, but there's no way to "attach" to a session and have an ongoing conversation with it, then switch to another.

## Proposed Solution

Add `/attach <session>` and `/detach` commands (or similar) that work like tmux:

```
/attach coding-projectA     # Route my messages to this session
> refactor the auth module  # Goes to projectA session
> what files did you change # Still in projectA context
/detach                     # Back to main session
> what's for dinner?     # Main session answers
/attach coding-projectA     # Resume where I left off
```

## Building Blocks Already Exist

- `sessions_spawn` with `cleanup: "keep"` creates persistent sessions
- `sessions_send` can send to other sessions
- `sessions_list` shows all sessions
- `sessions_history` reads session history

What's missing is the "attach" routing that redirects user messages to a different session until detach.
