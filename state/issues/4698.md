## Description

Title: Add reasoningDefault config option (like thinkingDefault / verboseDefault)

Body:

Problem

/reasoning level resets to off on every new session. There's no config default — unlike thinkingDefault and verboseDefault which have agents.defaults.* options.

The fallback chain in get-reply-directives.js:
const resolvedReasoningLevel = directives.reasoningLevel ??
    sessionEntry?.reasoningLevel ??
    "off";  // ← no agentCfg?.reasoningDefault
Compare with thinking/verbose which use agentCfg?.thinkingDefault / agentCfg?.verboseDefault.

Proposed solution

Add reasoningDefault to the config schema under agents.defaults (and per-agent override in agents.list[]):
{
  "agents": {
    "defaults": {
      "reasoningDefault": "on"
    }
  }
}
Valid values: "on" | "off" | "stream"

And update the resolution chain:
const resolvedReasoningLevel = directives.reasoningLevel ??
    sessionEntry?.reasoningLevel ??
    agentCfg?.reasoningDefault ??
    "off";

Context

Users who want reasoning visible by default (especially on Telegram with stream mode) currently need to send /reasoning on at the start of every session.
