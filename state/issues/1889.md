### @masterpaper (2026-01-27)

Hey! We hit a similar ARM compatibility issue on our Raspberry Pi 3B ‚Äî in our case it was the `@mariozechner/clipboard` module missing a native binary for `linux-arm-gnueabihf`.

We fixed it by patching the clipboard import to be optional (wrapping it in a try-catch). Full writeup and patch script here:
**https://github.com/masterpaper/clawdbot-raspberry-pi-fix**

For your Signal-cli issue, you might be able to apply a similar approach ‚Äî either patch the binary detection logic or manually download the correct ARM binary and swap it in.

Also worth noting: we needed `NODE_OPTIONS="--max-old-space-size=512"` to prevent memory crashes on Pi 3B.
