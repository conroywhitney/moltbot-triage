## Description

## Summary
Status card incorrectly reports "Activation: mention" for Always-on groups (Observed on Discord).

## Steps to reproduce
1. Configure a group channel (e.g. Discord) with `"requireMention": false` (Always on).
2. Run `/status` in that channel.

## Expected behavior
Status card should display `ðŸ‘¥ Activation: always-on`.

## Actual behavior
Status card displays `ðŸ‘¥ Activation: mention`.

## Environment
- Clawdbot version: 2026.1.30
- OS: Linux
- Install method: npm global
- Provider: Discord

## Suspected Cause
In `dist/auto-reply/status.js`, the `groupActivationValue` logic falls back to `"mention"` if the session entry doesn't have the value cached, ignoring the live config check.

```js
const groupActivationValue = isGroupSession
    ? (args.groupActivation ?? entry?.groupActivation ?? "mention") // <-- Bug: Defaults to "mention"
    : undefined;
```
