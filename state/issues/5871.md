## Description

## Summary

The `openclaw` command takes 17+ seconds to execute even for simple operations like `--help`, `--version`, or invalid commands. This makes the CLI nearly unusable.

## Steps to reproduce

1. Install openclaw on a Raspberry Pi 4
2. Run `time openclaw --help` or `time openclaw --version` or even an invalid command!
3. Observe the execution time

## Expected behavior

The `--help` and `--version` flags as well as most commandsshould execute in under 1 second.

### @kalinon (2026-02-02)

Additional Diagnostics

I did some profiling on this issue. Here are the findings:

Timing Breakdown
| Step                                      | Time                              |
| ----------------------------------------- | --------------------------------- |
| ./dist/infra/dotenv.js                    | 204 ms                            |
| ./dist/infra/env.js                       | 37 ms                             |
| ./dist/config/config.js                   | 5,025 ms ‚Üê 5+ seconds for config! |
| Plugin loading (killed before completion) | ~15+ seconds                      |
Total: 17-40 seconds for any CLI command, even --version

Root Causes

1. Config loading is pathologically slow ‚Äî 5+ seconds to parse openclaw.json (likely validating all model definitions with cost info)
2. Eager plugin loading ‚Äî AWS SDK, Google SDK, Slack SDK, Playwright, etc. all load on every command, not just when needed
3. ES module resolution overhead on ARM + SD card I/O (no SSD option on Pi 4)
4. Double startup penalty ‚Äî CLI respawns itself to set NODE_OPTIONS, paying the cost twice
Workarounds Tested

OPENCLAW_NO_RESPAWN=1 saves ~2-4 seconds but doesn't address the real issue.

Suggested Fix

Lazy-load plugin SDKs. Don't load AWS/Google/Slack/Playwright until those specific subcommands are actually invoked. This would cut CLI time to ~5-7 seconds (still slow, but usable).

Environment: Raspberry Pi 4, SD card (no SSD), 414MB node_modules
