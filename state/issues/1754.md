## Description


## Summary

Integrate [Cua's computer-server](https://github.com/trycua/cua/tree/main/libs/python/computer-server) into Clawdbot to enable desktop/screen control across all ExecHost types.

## Motivation

### Current Capabilities

Clawdbot agents can today:
- Run shell commands (in sandbox or gateway)
- Control browsers via CDP
- Read/write files
- Search the web
- Send messages

### What's Missing

Agents **cannot**:
- Take screenshots of the desktop
- Click at screen coordinates
- Type outside of browser contexts
- Control native applications
- See what's on screen (beyond browser)

This limits agents to CLI and browser workflows. With computer-use, agents could automate any GUI application.

## Proposed Solution

Add a `computer` tool that delegates to [cua-computer-server](https://pypi.org/project/cua-computer-server/), which provides 40+ desktop automation commands via WebSocket, HTTP, and MCP interfaces.

### How It Fits with ExecHost Types

| ExecHost | Current | With computer-server |
|----------|---------|---------------------|
| **Sandbox** | Docker container, shell + files | + desktop control via `cua-xfce`/`cua-ubuntu` images |
| **Node** | Companion app (iOS/Android/macOS) | + device screen control via embedded server |
| **Gateway** | Host shell + files | Not recommended (controls your actual screen) |

> **Note**: Computer-use on Gateway is discouraged since it gives agents control of your actual desktop. Use Sandbox (isolated container) or Node (separate device) instead.

### Architecture

```
┌─────────────────────────────────────────────────────────────┐
│                      Clawdbot Agent                         │
│  ┌───────────────────────────────────────────────────────┐  │
│  │                   Computer Tool                        │  │
│  │            (calls computer-server MCP/API)             │  │
│  └───────────────────────────────────────────────────────┘  │
└─────────────────────────────────────────────────────────────┘
                              │
              ┌───────────────┴───────────────┐
              ▼                               ▼
       ┌─────────────┐                 ┌─────────────┐
       │   Sandbox   │                 │    Node     │
       │  (Docker)   │                 │  (Device)   │
       │             │                 │             │
       │ cua-xfce or │                 │ computer-   │
       │ cua-ubuntu  │                 │ server      │
       │ (built-in)  │                 │ (embedded)  │
       └─────────────┘                 └─────────────┘
```

### End-User Setup

#### Sandbox Mode (recommended)

Just use Cua's desktop container images - no extra setup:

```bash
# In clawdbot config, set sandbox image to cua-xfce or cua-ubuntu
clawdbot configure
# → agents.defaults.sandbox.docker.image = "ghcr.io/trycua/cua-xfce:latest"
```

These images include computer-server running as a daemon with a virtual display.

#### Node Mode (companion devices)

Computer-server is embedded in the companion apps. Pair your device and agents can control it.

**Limitations:**
- **iOS**: Requires device to be on the same LAN as Clawdbot
- **Android**: May require ad-hoc setup (ADB, accessibility permissions)

### Available Sandboxes

All include computer-server pre-installed. See [Cua Desktop Sandboxes](https://cua.ai/docs/cua/guide/get-started/what-is-desktop-sandbox#docker-sandboxes) for details.

| Sandbox | OS | Description |
|---------|-----|-------------|
| `cua-xfce` | Linux | Docker + XFCE desktop |
| `cua-ubuntu` | Linux | Docker + Kasm desktop |
| `cua-qemu-linux` | Linux | QEMU/KVM VM (Ubuntu 22.04) |
| `cua-qemu-windows` | Windows | QEMU/KVM VM (Windows 11) |
| `cua-qemu-android` | Android | QEMU/KVM emulator |
| Lume | macOS | Native VM (Apple Silicon) |

### Computer Tool Actions

```typescript
interface ComputerToolInput {
  action:
    | 'screenshot'
    | 'click' | 'double_click' | 'right_click'
    | 'type' | 'key' | 'hotkey'
    | 'scroll' | 'move_mouse' | 'drag'
    | 'get_accessibility_tree'
    | 'get_screen_size';

  coordinate?: [number, number];
  text?: string;
  key?: string;
  keys?: string[];
  direction?: 'up' | 'down';
  amount?: number;
}
```

### Security Model

Follows Clawdbot's existing security patterns:

| ExecHost | Security | Approval |
|----------|----------|----------|
| **Sandbox** | Full access (isolated container) | None needed |
| **Node** | Device screen access | Per-device policy |

## Integration Approach

Since computer-server exposes an **MCP interface**, integration could be as simple as:

1. Connect to computer-server's MCP endpoint (`http://localhost:8000/mcp`)
2. Expose its tools as Clawdbot tools
3. Route based on ExecHost type

## Open Questions

1. **Tool exposure**: Single `computer` tool with action parameter, or separate tools?

2. **MCP vs custom**: Use computer-server's MCP directly, or build a custom adapter?
