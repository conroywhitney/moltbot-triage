## Description

## Feature Request

**What:**
Add support for the `safesearch` parameter when using Brave Search API via `web_search` tool.

**Why:**
Brave Search API supports a `safesearch` parameter with values: `strict`, `moderate`, `off`. Currently Clawdbot's `web_search` tool doesn't expose this, defaulting to Brave's default (`moderate`).

Users should be able to:
1. Set a default safesearch level in config (e.g., `tools.web.search.safesearch`)
2. Optionally override per-request via tool parameter

**Brave API docs:**
The `safesearch` query parameter controls filtering of adult/explicit content in results.

**Suggested implementation:**
- Add `safesearch` to config schema under `tools.web.search`
- Pass through to Brave API calls
- Optionally expose as tool parameter for per-query override
