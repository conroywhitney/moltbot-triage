## Description

# OpenClaw Bug Report: Legacy "clawdbot" References

## Issue Summary
OpenClaw contains multiple hardcoded references to "clawdbot" that cause module loading failures.

## Error Message
```
Error: Cannot find module 'clawdbot/plugin-sdk'
Require stack:
- C:\Users\vvvvv\AppData\Roaming\npm\node_modules\openclaw\extensions\minimax-portal-auth\index.ts
```

## Affected Files
1. **extensions/minimax-portal-auth/index.ts:1**
   - Current: `import { emptyPluginConfigSchema } from "clawdbot/plugin-sdk";`
   - Should be: `import { emptyPluginConfigSchema } from "openclaw/plugin-sdk";`

2. **dist/providers/github-copilot-utils.js:4**
   - Current: `return 'clawdbot/${version}';`
   - Should be: `return 'openclaw/${version}';`

3. **dist/protocol.schema.json**
   - Line 3: `"$id": "https://clawdbot.dev/protocol.schema.json"`
   - Line 60: `"const": "clawdbot-control-ui"`
   - Line 76: `"const": "clawdbot-macos"`
   - Line 92: `"const": "clawdbot-probe"`
   - All should use "openclaw" instead of "clawdbot"

## Environment
- OpenClaw version: Latest (installed via npm)
- Platform: Windows
- Installation: `npm install -g openclaw@latest`

## Impact
- Prevents MiniMax portal auth plugin from loading
- Affects other plugins that may depend on the SDK
- User agent string incorrectly identifies as "clawdbot"

## Reproduction Steps
1. Install openclaw: `npm install -g openclaw@latest`
2. Run: `openclaw config`
3. Select MiniMax OAuth option
4. Error appears: "Cannot find module 'clawdbot/plugin-sdk'"

## Suggested Fix
Perform a global search and replace of "clawdbot" â†’ "openclaw" across the entire codebase before publishing to npm.
