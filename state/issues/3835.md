## Description

## Summary
Add the ability to pin and name sessions to prevent auto-cleanup and improve discoverability.

## Motivation
- Sessions are auto-deleted after a period of inactivity
- Important sessions with valuable context get lost
- Session keys like `slack:thread:1769660721...` are not human-readable

## Proposed Features

### 1. Session Pinning
- Pinned sessions are excluded from auto-deletion
- Visible indicator in session list (e.g., ğŸ“Œ prefix)
- Commands: `/session pin`, `/session unpin`

### 2. Session Naming
- Assign custom human-readable names to sessions
- Example: `slack:thread:1769660721...` â†’ `"BTC Options Monitor Design"`
- Commands: `/session name <name>`, `/session rename <name>`

### 3. Web UI Integration
- **Session list in Control UI** with pin/unpin toggle
- **Inline name editing** in session list
- **Filter by pinned/named** sessions
- **Visual indicator** (ğŸ“Œ icon, custom name display)

## API Suggestions

```yaml
# Session metadata extension
sessions:
  pinned: boolean
  name: string | null
  pinnedAt: timestamp | null
```

thanks, until that time I did below config.patch update for my interactions-
DM sessions: long-but-finite idle timeout
session.dmScope = "main"
session.reset.mode = "idle"
session.reset.idleMinutes = 7200 (5 days)
So if you message again within 5 days, it should continue the same DM session/context; after ~5 days idle it will reset.

Session transcript indexing for recall
agents.defaults.memorySearch.enabled = true
agents.defaults.memorySearch.sources = ["memory", "sessions"]
So even if a session resets, I can still recall prior context by searching session transcripts.
