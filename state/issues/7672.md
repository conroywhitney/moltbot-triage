## Description


**_Some models do not support system role messages. When OpenClaw sends a system prompt to these models, they return no output at all (silent failure)._**

This is reproducible with Gemma3 12B and DeepSeek R1.

Steps to reproduce: 

- Configure OpenClaw to use Gemma3 12B or DeepSeek R1 via OpenRouter
- Send a simple user prompt (e.g. “Hello”)

Expected behavior:

If a model does not support system-role messages, OpenClaw should:

- Skip the system prompt, or
- **Merge the system prompt into the user message**, or
- Fail with a clear error explaining the incompatibility

Actual behavior:

- The model returns no response / empty output
- No error or warning is shown
- The agent appears stuck or broken

Environment:

Clawdbot version: 2026.1.30

OS: Pi OS (Raspberry Pi 5)

Install method (pnpm/npx/docker/etc): npm

Model backend: **OpenRouter**
