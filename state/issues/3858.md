## Description

## Summary

The npm package `moltbot` (version 0.1.0) is a placeholder package owned by a different party, not the actual Moltbot project. This causes users who follow documentation to install a non-functional package.

## Steps to reproduce

1. Run `npm install -g moltbot@latest`
2. Run `moltbot --version`
3. Get `zsh: command not found: moltbot`

## Expected behavior

`npm install -g moltbot` should install the actual Moltbot CLI, and `moltbot --version` should display the version.

## Actual behavior

- `npm install -g moltbot@latest` installs a placeholder package (version 0.1.0)
- This package has no `bin` field, so no CLI command is created
- Users get `command not found: moltbot`

The placeholder package contents:

\`\`\`json
{
    "name": "moltbot",
    "version": "0.1.0",
    "description": "moltbot",
    "keywords": ["moltbot"],
    "type": "module",
    "files": ["dist"],
    "engines": {"node": ">=24", "pnpm": ">=10"}
}
\`\`\`

## Environment

- Clawdbot version: N/A (wrong package installed)
- OS: macOS
- Install method: npm install -g moltbot@latest

## Logs or screenshots

\`\`\`bash
➜ npm install -g moltbot@latest

➜ moltbot --version

{

    ...
}
\`\`\`

## Suggested resolution

1. Acquire the `moltbot` npm package name from the current owner
2. Or update documentation to clarify the package name is `clawdbot`
3. Or publish under both names once the `moltbot` name is acquired
