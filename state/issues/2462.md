## Description

## Description
Automatic ack reactions (configured via `messages.ackReaction`) are not being applied to inbound Slack messages, even though manual reactions via the `message` tool work correctly.

## Config
```json
"messages": {
  "ackReaction": "üß†",
  "ackReactionScope": "all",
  "removeAckAfterReply": false
}
```

## Slack Scopes
Bot has `reactions:write` and `reactions:read` scopes.

## Observed Behavior
- Inbound DM messages do not receive the üß† ack reaction automatically
- Manual reactions via the `message` tool (action: react) work correctly
- No errors in logs related to ack reactions - they simply don't appear to fire
- Gateway restart did not resolve the issue

## Expected Behavior
Per the docs, with `ackReactionScope: "all"`, every inbound message should receive the configured ack reaction.

## Environment
- Clawdbot version: 2026.1.24-3
- Channel: Slack (socket mode)
- OS: macOS 26.2 (arm64)
- Node: v24.11.0

### @rossshannon (2026-01-30)

Slack uses a shortcode format for emojis. So whereas Telegram will accept the Unicode emoji ‚ÄúüëÄ‚Äù as the ack response, Slack requires the string `:eyes:`.

Works in Telegram:
```
  "messages": {
    "ackReaction": "üëÄ",
    "ackReactionScope": "all"
  }
```

Works in Slack
```
  "messages": {
    "ackReaction": ":eyes:",
    "ackReactionScope": "all"
  }
```
Fix in https://github.com/moltbot/moltbot/pull/4257
