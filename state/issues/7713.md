## Description

## Bug Description

After running `openclaw memory index`, the `Dirty` status in `openclaw memory status --deep` still shows `yes` when it should be `no`.

## Steps to Reproduce

1. Run `openclaw memory index`
2. Run `openclaw memory status --deep`
3. Observe `Dirty: yes` is still shown

## Expected Behavior

After indexing, the dirty status should be `no` (or `false`).

## Actual Behavior

Dirty status remains `yes` even after successful indexing.
