<!--StartFragment--><h1 id="bug-report-minimax-provider-missing-china-mainland-baseurl" class="atx">Bug Report: MiniMax Provider Missing China Mainland baseUrl</h1>
<h2 id="问题描述" class="atx">问题描述</h2>
<p>OpenClaw 的 MiniMax 模型提供商默认使用海外 baseUrl，但中国大陆用户需要使用国内专属端点，否则访问缓慢或无法连接。</p>
<h2 id="环境信息" class="atx">环境信息</h2>
<ul>
<li><strong>OS</strong>: Windows_NT 10.0.22631 (x64)</li>
<li><strong>OpenClaw 版本</strong>: 2026.1.x (latest)</li>
<li><strong>模型</strong>: minimax/MiniMax-M2.1</li>
</ul>
<h2 id="重现步骤" class="atx">重现步骤</h2>
<ol>
<li>运行 <code>openclaw configure</code></li>
<li>选择 <code>Model/auth</code> → <code>MiniMax M2.1</code></li>
<li>配置 API Key</li>
<li>尝试调用模型 → <strong>失败或超时</strong></li>
</ol>
<h2 id="预期行为" class="atx">预期行为</h2>
<p>中国用户应该能正常使用 MiniMax API。</p>
<h2 id="实际行为" class="atx">实际行为</h2>
<p>默认配置的 baseUrl 是海外地址，国内访问困难。</p>
<h2 id="根本原因" class="atx">根本原因</h2>
<p>OpenClaw 的 MiniMax provider 配置中，<strong>只定义了海外 baseUrl</strong>，没有国内端点选项。</p>
<h3 id="官方文档中的两个端点" class="atx">官方文档中的两个端点</h3>

地区 | baseUrl
-- | --
中国大陆 | https://api.minimaxi.com/anthropic
海外/默认 | https://api.minimax.io/anthropic


<blockquote>
<p>注意：<code>minimaxi</code> vs <code>minimax</code> — 国内端点是 <code>minimaxi.com</code>，海外是 <code>minimax.io</code></p>
<p>来源：MiniMax 官方 API 文档</p>
</blockquote>
<h2 id="修复建议" class="atx">修复建议</h2>
<h3 id="方案-1：时区自动检测（推荐）" class="atx">方案 1：时区自动检测（推荐）</h3>
<p>在 gateway 启动时检测用户时区，自动选择 baseUrl：</p>
<pre><code class="fenced-code-block language-javascript">// src/providers/minimax.ts (伪代码)

function getBaseUrlByRegion() {
  const region = Intl.DateTimeFormat().resolvedOptions().timeZone;

  const regionMap = {
    &#39;Asia/Shanghai&#39;: &#39;https://api.minimaxi.com/anthropic&#39;,
    &#39;Asia/Hong_Kong&#39;: &#39;https://api.minimaxi.com/anthropic&#39;,
    &#39;Asia/Taipei&#39;: &#39;https://api.minimaxi.com/anthropic&#39;,
    // 其他中国大陆时区...
  };

  return regionMap[region] || &#39;https://api.minimax.io/anthropic&#39;; // 默认海外
}</code></pre>
<h3 id="方案-2：配置文件支持区域字段" class="atx">方案 2：配置文件支持区域字段</h3>
<p>在 <code>openclaw.json</code> 中增加 <code>region</code> 配置：</p>
<pre><code class="fenced-code-block language-json">{
  &quot;models&quot;: {
    &quot;providers&quot;: {
      &quot;minimax&quot;: {
        &quot;region&quot;: &quot;cn&quot;,  // 或 &quot;global&quot;
        &quot;baseUrl&quot;: &quot;${auto}&quot;  // 根据 region 自动选择
      }
    }
  }
}</code></pre>
<h3 id="方案-3：文档增强（最低成本）" class="atx">方案 3：文档增强（最低成本）</h3>
<p>在 <code>docs/providers/minimax.md</code> 中添加提示：</p>
<pre><code class="fenced-code-block language-markdown">## 中国大陆用户

国内用户需要在配置中手动指定 baseUrl：

```json
{
  &quot;models&quot;: {
    &quot;providers&quot;: {
      &quot;minimax&quot;: {
        &quot;baseUrl&quot;: &quot;https://api.minimaxi.com/anthropic&quot;
      }
    }
  }
}</code></pre>
<p>然后运行 <code>openclaw gateway restart</code> 重启。</p>
<pre><code class="fenced-code-block">## 工作around（临时方案）

用户可以手动编辑 `openclaw.json`：

```json
{
  &quot;models&quot;: {
    &quot;providers&quot;: {
      &quot;minimax&quot;: {
        &quot;baseUrl&quot;: &quot;https://api.minimaxi.com/anthropic&quot;,
        &quot;apiKey&quot;: &quot;sk-...&quot;,
        &quot;api&quot;: &quot;anthropic-messages&quot;,
        &quot;models&quot;: [...]
      }
    }
  }
}</code></pre>

</ul>
