## Description

## Problem

When using the Control UI webchat, there's no visible indicator showing which model is currently active for the session. This makes it difficult to confirm which model is handling responses, especially when switching between multiple models (e.g., local LLM via LM Studio vs cloud models like Claude).

## Current workarounds

- Type `/status` in chat (consumes tokens, requires agent response)
- Run `clawdbot status` from CLI (requires terminal access)
- Check the Sessions tab (requires switching views)

## Proposed solution

Add a persistent, visible indicator in the webchat interface showing the current model. Options:

- **Header badge**: Small text/pill showing model alias or name (e.g., `local` or `lmstudio/gpt-oss-20b`)
- **Status bar**: Thin bar under the header with model + context info
- **Input area label**: Small text above or below the chat input

## Why it matters

- Users switching between local and cloud models need quick confirmation
- Debugging model-specific behavior is easier when the model is visible
- Reduces unnecessary `/status` calls that consume tokens
