## Description

### Describe the bug
When configuring the Minimax provider in `clawdbot.json` with a custom `baseUrl` (to fix the incorrect hardcoded `.io` domain), the configuration is ignored if `models.json` is missing or fails to generate.

It appears the runtime falls back to an in-memory implicit provider resolution (`resolveImplicitProviders`) which uses the hardcoded default values and **skips the merge logic** (`mergeProviders`) that would otherwise apply the user's config.

### To Reproduce
1.  Remove `models.json` from the agent workspace.
2.  Configure `minimax` in `clawdbot.json` with the correct URL:
    ```json
    "models": {
      "providers": {
        "minimax": { "baseUrl": "https://api.minimaxi.com/anthropic" }
      }
    }
    ```
3.  Run an agent using `minimax/MiniMax-M2.1`.
4.  Observe the 401 error.
5.  Intercept the fetch call (or check logs) to see it still hits `https://api.minimax.io` (the hardcoded default).

### Expected behavior
The user configuration in `clawdbot.json` should always override internal defaults, even when `models.json` is regenerated or when falling back to implicit resolution.

### Technical Details
- **Hardcoded Error**: `dist/agents/models-config.providers.js` defines `MINIMAX_API_BASE_URL` as `.io` instead of `.com`.
- **Logic Gap**: When `ensureClawdbotModelsJson` fails to write or the runtime falls back, `resolveModel` likely uses the raw implicit provider without merging `cfg.models.providers`.

### Environment
- Clawdbot Version: 2026.1.24-3
- Model: Minimax-M2.1
