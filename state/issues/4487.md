## Description

## Use Case

When the machine is off (or the gateway isn't running) at the time a scheduled cron job should fire, the job is silently skipped. For one-shot reminders, this means they're lost entirely.

## Proposed Behavior

On gateway startup, check all cron jobs:
- Compare `lastRun` (or last known execution time) with the job's schedule
- If a job *should* have fired while the gateway was down, trigger it now (or mark it for immediate execution)

This is similar to how `anacron` handles missed jobs on Linux systems.

## Configuration

Could be opt-in per job or global:
- `catchUp: true` on individual jobs
- Or a global `cron.catchUpMissed: true` setting

## Notes

Particularly useful for:
- One-shot reminders (currently lost if machine is off)
- Daily tasks that should always run, even if the machine was off overnight
