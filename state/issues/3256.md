## Description

## Feature Request

### Problem
When using `voice_call` in `conversation` mode, the intro message is played to the recipient via TTS, but the conversation AI doesn't receive this context. This means the AI doesn't know the purpose of the call.

### Example
When I call someone with:
```
message: "Hej Ole! Henrik bad mig ringe og spÃ¸rge hvordan din mor har det."
```

The recipient hears this, but when they respond, the AI starts fresh and asks generic questions like "How are you?" instead of following up on the stated purpose (asking about the mother).

### Expected Behavior
The intro message should be included in the conversation AI's context/system prompt, so it knows the purpose of the call and can follow up appropriately.

### Current Workaround
Include all context directly in the intro message so the recipient hears the full question, and repeat key info so the AI picks it up from the transcript.

### Suggested Implementation
- Add the intro message to the `responseSystemPrompt` dynamically for each call
- Or include it as the first message in the conversation history

### Environment
- Clawdbot version: 2026.1.24-3
- Plugin: @clawdbot/voice-call
- Provider: Twilio
