## Description

## Summary

OpenClaw incorrectly detects missing npm global in PATH. It says there is no global bin in PATH despite it's already there:

```
→ PATH warning: missing npm global bin dir: /home/ubuntu/.npm-global/bin
This can make openclaw show as "command not found" in new terminals.
Fix (zsh: ~/.zshrc, bash: ~/.bashrc):
  export PATH="/home/ubuntu/.npm-global/bin:\/home/ubuntu/.npm-global/bin:/usr/local/sbin:/usr/local/bin:/usr/sbin:/usr/bin:/sbin:/bin:/usr/games:/usr/local/games:/snap/bin"
```

You can see `\/home/ubuntu/.npm-global/bin` is already there, I think it missed the `\`.

I think

## Steps to reproduce
1. Create an AWS EC2 Ubuntu instance
2. Copy and paste the OpenClaw install command `curl -fsSL https://openclaw.ai/install.sh | bash`
3. Wait until this appears

## Expected behavior

It should be able to detect \/home/ubuntu/.npm-global/bin` in `PATH`

## Actual behavior

It failed to detect it with `\` escape character.

## Environment
- OpenClaw version: 2026.1.29
- OS: Ubuntu 24
- Install method (pnpm/npx/docker/etc): `curl -fsSL https://openclaw.ai/install.sh | bash`
