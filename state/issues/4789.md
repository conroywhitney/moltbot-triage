## Description

## Environment
- **OS:** Windows 11 (10.0.26200)
- **Clawdbot:** 2026.1.24-3
- **gog:** v0.9.0 (99d9575 2026-01-22)
- **Node:** v24.13.0

## Problem
`clawdbot webhooks gmail setup --account <email>` fails with:
```
Error: gog not installed; install it and retry
```

However, gog is installed and works correctly from the same shell:
```powershell
PS> gog --version
v0.9.0 (99d9575 2026-01-22T04:15:12Z)

PS> where.exe gog
C:\Windows\System32\gog.exe
C:\Users\NickLopez\dev\tools\gogcli\gog.exe
```

## Investigation
1. `gog --version` works fine from PowerShell
2. `gog.exe` exists in `C:\Windows\System32` (and a second copy elsewhere)
3. The wizard's gog detection still fails
4. Tried `tools.exec.pathPrepend` but discovered a separate bug (see below)

## Related Bug: tools.exec.pathPrepend replaces PATH instead of prepending

When setting:
```json
{tools:{exec:{pathPrepend:[C:\\Windows\\System32,C:\\Users\\NickLopez\\dev\\tools\\gogcli]}}}
```

The exec environment's PATH becomes *only* those two directories, instead of prepending to the existing PATH:
```powershell
PS> $env:PATH
C:\Windows\System32;C:\Users\NickLopez\dev\tools\gogcli
```

This breaks other tools (including `clawdbot` itself) since the rest of PATH is gone.

## Expected Behavior
1. `clawdbot webhooks gmail setup` should detect gog when it's in PATH on Windows
2. `tools.exec.pathPrepend` should prepend to PATH, not replace it

## Workaround
None found for the gog detection issue. Manual setup per docs works but requires more steps.
