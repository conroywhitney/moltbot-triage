## Description

### Description
When setting `gateway.bind` to `"lan"` or `"auto"`, the gateway still binds only to `127.0.0.1` and `[::1]` instead of the LAN interface IP.

### Steps to Reproduce
1. Set `gateway.bind: "lan"` (or `"auto"`) in `clawdbot.json`
2. Restart the gateway
3. Check logs

### Expected Behavior
Gateway should listen on the LAN IP (e.g., `192.168.1.177:18789`)

### Actual Behavior
Gateway logs show:
```
[gateway] listening on ws://127.0.0.1:18789
[gateway] listening on ws://[::1]:18789
```

LAN IP is unreachable:
```
curl localhost:18789     → 200 ✓
curl 192.168.1.177:18789 → connection refused ✗
```

### Environment
- **OS:** macOS (Darwin 25.2.0 arm64)
- **Clawdbot:** 2026.1.24-3
- **Node:** v23.8.0

### Workaround
Using `socat` to proxy:
```bash
socat TCP-LISTEN:18789,bind=192.168.1.177,fork TCP:127.0.0.1:18789
```
