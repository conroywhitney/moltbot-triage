## Description

## Summary
The **Clawdbot.app** on macOS consistently crashes with an `EXC_BAD_INSTRUCTION` error when attempting to run the `peekaboo` screenshot command via the lookback gateway. The crash is triggered during the initialization of the `CodeTokenizer` component.

## Steps to reproduce
1. Set up **Clawdbot.app** on macOS as a node using the **lookback gateway** configuration.
2. Instruct the bot to launch an application (e.g., **VS Code**).
3. Request the bot to perform a screenshot using the **peekaboo** tool.
4. The bot attempts to execute the command, and the macOS application (`Clawdbot.app`) crashes immediately.

## Expected behavior
The bot should capture the screenshot via the Peekaboo bridge and continue operation without crashing.

## Actual behavior
The application terminates abruptly. Based on the crash report, a Swift assertion fails when the app tries to initialize syntax highlighting/tokenization resources.

## Environment
- **Clawdbot version**: 2026.1.22 (Build 7530)
- **OS**: macOS 15.2 (Build 25C56)
- **Hardware**: MacBookPro16,1 (Intel/X86-64)
- **Install method**: .app (Applications folder)
