## Description

## Description

The webchat UI has form elements without `id` or `name` attributes, triggering browser accessibility warnings:

> A form field element has neither an id nor a name attribute. This might prevent the browser from correctly autofilling the form.

## Affected elements

1. **Message textarea**: The main input textarea (placeholder: "Message...")
2. **Session selector**: `.chat-controls__session select`

## Suggested fix

Add unique id attributes:
- `id="webchat-message-input"` for the textarea
- `id="webchat-session-select"` for the session dropdown

## Why it matters

- Improves accessibility for screen readers
- Enables proper browser autofill behavior
- Eliminates DevTools warnings

## Additional enhancement suggestions

While fixing the id/name attributes, consider these UX improvements for the message textarea:

### 1. Auto-resizing textarea
The fixed height with manual resize is clunky for a chat input. Auto-expand based on content:

```js
textarea.addEventListener('input', () => {
  textarea.style.height = 'auto';
  textarea.style.height = textarea.scrollHeight + 'px';
});
```

Respects the existing `max-height: 150px` constraint.

### 2. Focus state visibility
Currently has `outline: none` which hurts accessibility. Add a visible focus indicator:

```css
textarea:focus {
  border-color: var(--accent);
  box-shadow: 0 0 0 2px var(--accent-subtle);
}
```

### 3. Accessibility
- Ensure the label provides sufficient context for screen readers
- Consider `aria-label` if the visual label is hidden/icon-only

These are nice-to-haves on top of the core id/name fix.
