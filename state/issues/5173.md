## Description

## Summary

在使用Clawdbot（后更名为Moltbot）+MiniMax 2.1模型时，国内用户将Base URL调整为国内配置后，Web UI无法与模型对话，界面无输出，终端显示no output。
初步判断问题与gateway 启动方式（前台/后台 ）以及旧版本 Clawdbot的稳定性有关。

后续在升级至Moltbot 2026.1.26 后，问题基本解决。

## Steps to reproduce

问题复现（旧版本 Clawdbot）
1. 使用Clawdbot 2026.1.24-3
2. 在Web UI中配置MiniMax 2.1，将Base URL改为国内用户配置，Auth Header打开
3. 使用后台方式启动gateway：clawdbot gateway start
4. 打开或刷新 Web UI
5. 尝试与模型对话

## Expected behavior
- Web UI可以正常通MiniMax 2.1进行对话（返回正确回复内容）
- gateway后台运行稳定，不影响Web UI/channel连接

## Actual behavior
问题表现： 
- Web UI中对话无任何输出
- 终端显示no output

临时解决方法（旧版本Clawdbot）：
- 必须以前台方式启动 gateway 才能正常对话
clawdbot gateway stop
clawdbot gateway --verbose
- 刷新Web UI后，可正常与MiniMax 2.1对话

遗留问题（旧版本Clawdbot）：
1. 切换回后台启动（LaunchAgent）后，Web UI仍无法对话
2. 在前台 verbose 模式下：一旦新增 channel（如 Telegram），gateway日志窗口会中断，模型掉线
3. Telegram bot中输入/start无任何响应
4. 不确定是本地macOS环境问题，还是Clawdbot本身的bug

## Update
Clawdbot 因商标原因已正式更名为 Moltbot。通过以下方式安装最新版本后，问题基本解决：

pnpm moltbot onboard --install-daemon

- 新版本：🦞 Moltbot 2026.1.26
- 旧版本：🦞 Clawdbot 2026.1.24-3

**当前可行方案**
- 在 Web UI 中将 MiniMax 2.1 设置为国内用户配置
- 使用前台方式启动 gateway（可实时查看日志）：
pnpm moltbot gateway run
- 刷新 Web UI，即可正常对话

目前该方式下Web UI与模型交互稳定，后续也有社区教程基于该版本成功接入telegram、飞书等channel。整体来看，随着 Moltbot 版本更新，gateway稳定性和可用性明显提升。

**解决的其他问题**
【问题1：Telegram端配对码接收问题】Telegram robot \start 未正常发送配对码。
【解决方法】🪜用TUN模式即可解决（还不行就换节点试试）。
	
【问题2：终端配对码报错问题】终端窗口输入pnpm moltbot pairing approve telegram XXX（配对码），返回报错：Failed to start CLI: Error: Channel telegram does not support pairing
【解决方法】如图所示，在Web UI界面添加你的Telegram ID号（robot发送的配对信息里的user id，不是Pairing code）。
【具体操作】Web UI ➡️ Config ➡️ Channels ➡️ Allow From点击+Add按钮，在#1下面的框里输入user id ➡️ 界面右上角Save+Update （一直保持gateway run）
	
心得：上述问题包括之前解决的Moltbot连接MiniMax后no output问题，我把问题详细描述后抛给Grok和ChatGPT都不能很好的解决，越改越麻烦，怕给系统环境改坏了又没法复原。后来甚至尝试过在VS Code里用同义灵码的Agent帮我把MiniMax输出跑通，它对json文件一顿改，但最后还是没跑通（no output）。个人感觉还是Moltbot比较新，这些模型还没有学到调通他的知识。像本文解决的问题在网上藏得很深，手动搜索还是能找到共鸣和大神指点，还得是人类会学习:)。Moltbot的Web UI做的不错，避免了修改json文件看源码的麻烦。

Telegram连接问题的解决感谢：https://github.com/openclaw/openclaw/issues/3793
