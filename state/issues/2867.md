## Bug Report

### Description
In the Moltbot webUI, long setting pages (e.g., Settings/config â†’ Authentication or Agents) do not scroll properly. The content gets cut off and users cannot access all settings.

### Steps to Reproduce
1. Open Moltbot webUI
2. Go to Settings/config
3. Navigate to Authentication or Agents section
4. Try to scroll down to see all settings
5. The content is cut off and cannot be scrolled

### Root Cause
In `/ui/src/styles/config.css`:
- `.config-layout` uses `min-height` instead of `height`, causing overflow issues
- `.config-main` lacks `min-height: 0`, preventing flex children from shrinking properly

### Fix
Change in `/private/tmp/moltbot/ui/src/styles/config.css`:

1. Replace line ~13:
```css
/* Before */
min-height: calc(100vh - 140px);

/* After */
height: calc(100vh - 140px);
```

2. Add `min-height: 0` to `.config-main`:
```css
.config-main {
  display: flex;
  flex-direction: column;
  min-width: 0;
  min-height: 0;  /* Add this line */
  background: var(--panel);
}
```

3. Update mobile responsiveness media query (~line 1292):
```css
@media (max-width: 768px) {
  .config-layout {
    grid-template-columns: 1fr;
    height: auto;
    min-height: calc(100vh - 140px);
    overflow: visible;
  }
}
```

### Environment
- Clawdbot version: 2026.1.24-0
- Browser: Any modern browser
- Screen size: Desktop and mobile affected

### Priority
Medium - affects usability of settings pages with many options
