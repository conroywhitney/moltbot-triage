## Description

## Bug Description
When loading a plugin extension from `~/.clawdbot/extensions/`, if the `clawdbot.plugin.json` manifest is missing the `configSchema` field, the gateway crashes with:

```
ERROR: plugin manifest requires configSchema
(/home/nuc/.clawdbot/extensions/discord-voice/clawdbot.plugin.json)
```

## Expected Behavior
The gateway should either:
1. Gracefully handle missing configSchema (default to empty object schema)
2. Log a warning and skip the plugin instead of crashing
3. Provide a clearer error message about what's needed

## Steps to Reproduce
1. Create a plugin in `~/.clawdbot/extensions/my-plugin/`
2. Add a `clawdbot.plugin.json` without `configSchema` field:
```json
{
  "id": "my-plugin",
  "name": "My Plugin",
  "version": "0.1.0",
  "main": "dist/index.js"
}
```
3. Restart gateway
4. Gateway crashes

## Environment
- Clawdbot version: 2026.1.24-3
- OS: Ubuntu 24.04 (Linux 6.8.0-90-generic)
- Node: v22.22.0

## Workaround
Add a configSchema to the manifest:
```json
{
  "configSchema": {
    "type": "object",
    "additionalProperties": false,
    "properties": {}
  }
}
```
