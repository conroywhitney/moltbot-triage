## Description

## Description
When using Groq as the model provider (e.g. `/model groq/llama-3.3-70b-versatile`), the embedded agent starts and completes immediately (~25-80ms) without producing any response.

## Steps to reproduce
1. Configure `GROQ_API_KEY` in env.vars
2. Add `groq:default` auth profile with mode=token
3. Switch model: `/model groq/llama-3.3-70b-versatile`
4. Send any message

## Expected behavior
Agent should respond using Groq/Llama

## Actual behavior
- Run starts and ends in 25-80ms
- No tool calls, no response
- `run_completed` but nothing sent to user

## Logs
```
embedded run start: provider=groq model=llama-3.3-70b-versatile
embedded run agent start
embedded run agent end  (25-80ms later)
embedded run prompt end durationMs=57
```

## Notes
- The API key works fine with direct curl calls
- Claude/Anthropic works normally on the same session
- Version: 2026.1.24-3
