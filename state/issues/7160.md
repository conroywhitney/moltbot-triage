## Description

## Summary
File: src/agents/tools/gateway.ts:14
The callGatewayTool function has a hardcoded 10-second timeout for node commands, which is too short for common real-world use cases like IMAP email fetching, package installations, API calls, and script execution.




## Steps to reproduce

1.Run a long-running script on a node:
nodes.run({
  node: "Mac Node (headless)",
  command: ["python3", "/path/to/gmail_digest.py"],
  commandTimeoutMs: 60000,
  invokeTimeoutMs: 60000
})


2. Observe that the command times out after exactly 10 seconds:
gateway timeout after 10000ms

3.The commandTimeoutMs parameter is ignored

## Expected behavior

Default timeout should be 60 seconds (or higher)
• commandTimeoutMs parameter should be respected
• OR add config option: gateway.nodes.defaultCommandTimeoutMs
Actual behavior

Timeout is hardcoded at 10 seconds in src/agents/tools/gateway.ts:

const timeoutMs =
  typeof opts?.timeoutMs === "number" && Number.isFinite(opts.timeoutMs)
    ? Math.max(1, Math.floor(opts.timeoutMs))
    : 10_000; // <-- Hardcoded 10s
