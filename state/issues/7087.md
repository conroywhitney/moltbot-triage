## Description

### 问题描述
通过OpenClaw Web界面更改定时任务的启用/禁用状态时，操作似乎无效。即使在界面上切换了状态，实际配置文件中的状态并未相应更新，或者界面显示与实际状态不符。

### 重现步骤
1. 创建或选择一个定时任务
2. 通过Web界面将定时任务设置为禁用（enabled: false）
3. 再次通过Web界面尝试将其设置为启用（enabled: true）
4. 检查配置文件 `/Users/dewly/.openclaw/cron/jobs.json`
5. 观察定时任务是否真正被启用

### 实际结果
- 通过Web界面更改状态后，配置文件中的`enabled`字段可能未被正确更新
- 或者界面显示的状态与实际配置文件中的状态不一致
- 定时任务可能仍然保持原状态（例如，仍然是禁用状态）

### 期望结果
- 通过Web界面更改定时任务状态应正确更新配置文件中的`enabled`字段
- 界面显示的状态应与配置文件中的实际状态保持一致
- 启用的定时任务应按照设定的时间间隔正常执行

### 工作区解决方案
目前发现可以通过直接编辑配置文件来解决此问题：
```bash
# 编辑定时任务配置文件
vi ~/.openclaw/cron/jobs.json
# 手动将 "enabled": false 改为 "enabled": true
```

### 环境信息
- OpenClaw版本: 2026.1.30 (build 8469)
- 运行模式: local
- 操作系统: macOS Darwin 25.2.0 (arm64)
- 浏览器: Chrome 版本 144.0.7559.110（正式版本） (arm64)

### 附加信息
- 直接编辑JSON配置文件可以正常更改定时任务状态
- 定时任务功能本身工作正常，只是Web界面控制存在问题
- 可能涉及Web界面与后端API通信或配置文件写入的问题
