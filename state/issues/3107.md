## Description

## Summary

   When using `clawdbot tui --url <remote-gateway>` as a pure
 client, config validation still fails if local plugins are
 not installed.

   ## Steps to Reproduce

   1. Install clawdbot on a machine intended only as a TUI
 client
   2. Run: `clawdbot tui --url ws://remote-gateway:port
 --token "xxx"`
   3. Observe validation error about plugins.slots.memory

   ## Attempted Workarounds (all fail)

   | Config | Result |
   |--------|--------|
   | No config file | Defaults apply, memory-core not found |
   | `{}` | Same |
   | `{"plugins":{"enabled":false}}` | Still validates
 default memory slot |
   | `{"plugins":{"slots":{"memory":null}}}` | Schema rejects
 null |

   ## Root Cause

   1. `normalizePluginsConfig()` always defaults `memory:
 "memory-core"`
   2. Validation checks if plugin exists locally
   3. `tui` not in `ALLOWED_INVALID_COMMANDS`

   ## Expected

   - Add `tui` to allowed commands when `--url` is provided
   - Or allow empty string to disable slot
   - Or skip plugin validation for client-only commands
