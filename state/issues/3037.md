## Description

## Summary
Running a self-hosted [Telegram Bot API server](https://github.com/tdlib/telegram-bot-api) unlocks higher file limits, lower latency, and more webhook connections. grammY supports this via `apiRoot`, but Clawdbot's Telegram plugin doesn't expose the option.

## Proposed solution
Add an `apiRoot` config option to the Telegram channel config:

```json
{
  "channels": {
    "telegram": {
      "apiRoot": "http://localhost:8081"
    }
  }
}
```

And pass it to grammY's Bot constructor in `dist/telegram/bot.js`:

```typescript
const apiRoot = telegramCfg?.apiRoot;
const bot = new Bot(opts.token, {
  ...(client && { client }),
  ...(apiRoot && { client: { ...client, apiRoot } })
});
```

## Alternatives considered
- Environment variable (e.g. `TELEGRAM_API_ROOT`) — grammY doesn't support this natively
- Patching the plugin locally — works but not maintainable

## Additional context
- [grammY apiRoot docs](https://grammy.dev/ref/core/apiclientoptions#apiroot)
- [Telegram Bot API server repo](https://github.com/tdlib/telegram-bot-api)
- [Local Bot API benefits](https://core.telegram.org/bots/api#using-a-local-bot-api-server): uploads up to 2GB, no download limits, local file paths, up to 100k webhook connections
- Clawdbot version: 2026.1.23-1
