## Description

## Bug Report

**Clawdbot version:** 2026.1.24-3

**Issue:** Venice AI provider configured correctly but produces empty responses. No API calls are being made to Venice endpoint.

### Configuration

```json
{
  "models": {
    "mode": "merge",
    "providers": {
      "venice": {
        "baseUrl": "https://api.venice.ai/api/v1",
        "apiKey": "VENICE-INFERENCE-KEY-***",
        "auth": "api-key",
        "api": "openai-completions",
        "models": [
          {
            "id": "llama-3.3-70b",
            "name": "Llama 3.3 70B",
            "reasoning": false,
            "input": ["text"],
            "cost": {"input": 0, "output": 0, "cacheRead": 0, "cacheWrite": 0},
            "contextWindow": 200000,
            "maxTokens": 8192
          }
        ]
      }
    }
  },
  "agents": {
    "defaults": {
      "model": {
        "primary": "anthropic/claude-sonnet-4-5",
        "fallbacks": ["venice/deepseek-v3.2", "venice/llama-3.3-70b"]
      }
    }
  }
}
```

### Expected Behavior
Venice models should make API calls and return completions.

### Actual Behavior
- Agent runs complete in ~650ms
- No HTTP requests logged to Venice API
- No errors in logs
- Empty responses (no output)
- Direct curl to Venice API with same credentials **works fine**

### Test Evidence

Spawned multiple test agents with Venice models:
```
model=deepseek-v3.2, runtime=1617ms → empty output
model=llama-3.3-70b, runtime=1569ms → empty output
```

Log excerpt showing no API activity:
```
embedded run start: provider=venice model=deepseek-v3.2
embedded run agent start
embedded run agent end
embedded run done: durationMs=1617 aborted=false
No reply from agent.
```

### Direct API Test (Works)
```bash
curl -X POST "https://api.venice.ai/api/v1/chat/completions" \
  -H "Authorization: Bearer [REDACTED]***" \
  -d '{"model":"llama-3.3-70b","messages":[{"role":"user","content":"test"}],"max_tokens":10}'
# Returns valid response in ~1.5s
```

### Diagnosis
The Venice provider appears to initialize but never actually invokes the API. Likely issue with the `openai-completions` adapter for Venice, or provider registration isn't happening.

### Environment
- OS: Linux (Debian)
- Node: v22.22.0
- Gateway mode: local
- Venice models discovered: timeout (using static catalog)
