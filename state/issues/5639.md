## Description

## Description

iOS app connects successfully as a node and shows `paired · connected` status with capabilities `canvas, screen`, but all `node.invoke` commands timeout without response.

## Environment

- **iOS app version:** 2026.1.24 (also tested 2026.1.27-beta.1 and 2026.1.30)
- **Gateway version:** 2026.1.24-3 (npm)
- **Device:** iPhone 18,3 (iPhone 17)
- **iOS version:** 26.2.1
- **Gateway host:** macOS (Mac Studio)

## Steps to Reproduce

1. Build iOS app from source (`v2026.1.24` tag)
2. Connect to gateway (shows `Connected` in app, `paired · connected` in `clawdbot nodes status`)
3. Run any node invoke command:
   ```bash
   clawdbot nodes invoke --node iPhone --command canvas.present --params '{"url":"https://example.com"}'
   ```
4. Command times out after ~15-30 seconds

## Expected Behavior

Canvas should display the URL on the iOS device.

## Actual Behavior

```
nodes invoke failed: Error: node invoke timed out
```

Gateway logs show:
```
[ws] ⇄ res ✗ node.invoke 15002ms errorCode=UNAVAILABLE errorMessage=node invoke timed out
```

## Additional Context

- App UI shows `Connected` with green indicator
- `clawdbot nodes describe` shows correct capabilities and commands
- Tested with voice/camera features disabled
- Tested multiple app versions (1.24, 1.27-beta, 1.30 - all same behavior)
- **macOS node works perfectly** with same gateway - confirms gateway-side is fine
- v2026.1.30 iOS app crashed on launch with `NSInvalidArgumentException` (separate issue?)

## Xcode Console

Only shows iOS system noise (RTIInputSystemClient keyboard messages), no app-level errors when invoke is attempted.
