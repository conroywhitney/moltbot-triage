## Description

## Bug Description

When a tool like `session_status` is called with no parameters, the toolCall entry is sometimes saved to the session JSONL without an `arguments` or `input` field:

```json
{
  "type": "toolCall",
  "id": "toolu_...",
  "name": "session_status"
  // MISSING: "arguments": {}
}
```

When this session is later loaded as conversation history, the Anthropic API rejects it with:

```
LLM request rejected: messages.27.content.1.tool_use.input: Field required
```

## Expected Behavior

Tool calls with no parameters should still include an empty `arguments: {}` field to satisfy the API schema.

## Reproduction

1. Call `session_status` (or another tool with all-optional params) with no arguments
2. Session gets saved
3. On next conversation turn, API rejects the malformed history

## Workaround

Manually move corrupted session files out of the sessions directory:
```bash
mkdir -p ~/.openclaw/agents/main/sessions/_corrupted
# Find and move files with toolCall entries missing arguments
```

## Environment

- OpenClaw version: 2026.1.30
- OS: macOS
- Provider: google-antigravity (Claude via Anthropic API)
