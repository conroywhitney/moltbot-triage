## Description


## Environment

- **Version:** 2026.1.30
- **OS:** Windows 11
- **Node:** 22.x
- **Provider:** minimax-portal/MiniMax-M2.1
- **Gateway Mode:** local

## Summary

MiniMax-M2.1 responses show "(no output)" in the TUI while the response is correctly received and displayed in the GUI dashboard. The model generates valid responses but OpenClaw fails to parse and display them.

## Configuration

```json
{
  "id": "MiniMax-M2.1",
  "name": "MiniMax M2.1",
  "reasoning": false,
  "api": "anthropic-messages"
}
```

## Expected Behavior

MiniMax responses should be displayed in the TUI, just like Claude CLI responses.

## Actual Behavior

- TUI shows: `(no output)`
- GUI Dashboard displays the response correctly
- API calls succeed and return valid JSON

## Reproduction Steps

1. Configure MiniMax-M2.1 as the model
2. Send a message via TUI
3. Observe "(no output)" in TUI
4. Check GUI Dashboard - response appears there

## Root Cause

In `dist/utils/provider-utils.js`, the function `isReasoningTagProvider()` returns `true` for MiniMax:

```javascript
// Handle Minimax (M2.1 is chatty/reasoning-like)
if (normalized.includes("minimax")) {
    return true;
}
```

This causes OpenClaw to expect reasoning tags (`<final>`) in the text stream, but MiniMax returns JSON with a `reasoning_content` field instead:

```json
{
  "choices": [{
    "message": {
      "content": "Bonjour ! ðŸ‘‹",
      "reasoning_content": "The user is asking me to say 'bonjour'..."
    }
  }]
}
```

OpenClaw's parser fails to extract the `content` field and shows "(no output)".

## Session Log Evidence

Direct API call to MiniMax returns:
```json
{
  "choices": [{
    "finish_reason":"stop",
    "message":{
      "content":"Bonjour ! ðŸ‘‹",
      "reasoning_content":"The user is asking me to say \"bonjour\" (hello in French)...",
      "reasoning_details":[{"type":"reasoning.text",...}]
    }
  }],
  "usage":{"completion_tokens_details":{"reasoning_tokens":37}}
}
```

## Impact

- MiniMax models are unusable in TUI mode
- Users must use GUI to see responses
- This affects all MiniMax model variants

## Suggested Fix

Remove MiniMax from `isReasoningTagProvider()` or update the parsing logic to handle MiniMax's JSON format with `reasoning_content` field properly.
