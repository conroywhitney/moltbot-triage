## Description

## Summary
`moltbot gateway install --force` flag is ignored, and `moltbot status` reports gateway as "running" while crash-looping with port unreachable.

## Steps to reproduce

**Issue A: --force ignored**
1. Have an existing (possibly broken) gateway service installed
2. Run `moltbot gateway install --force`
3. Observe it tells you to use `--force` (which you already did)

**Issue B: False "running" status**
1. Have a config mismatch (e.g., `gateway.bind=tailnet` with Tailscale Serve mode)
2. Service crash-loops due to config error
3. Run `moltbot status`

## Expected behavior

**A:** `--force` should remove and reinstall the service

**B:** Status should show "crash-looping" or "failed" when service isn't actually listening on port

## Actual behavior

**A:**
```
$ moltbot gateway install --force
Gateway service already enabled.
Reinstall with: moltbot gateway install --force
```

**B:** Status shows "running (pid X, state active)" while gateway is unreachable and journalctl shows restart counter at 65.

## Environment
- Moltbot version: 2026.1.24-3
- OS: Ubuntu 24.04 (DigitalOcean 2GB)
- Install method: pnpm
