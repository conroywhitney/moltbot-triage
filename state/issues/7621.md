## Description

## Description

When trying to switch to the `minimax-cn/MiniMax-M2.1` model (alias `mm-cn`), the system throws a 401 authentication error.

## Error Message

```
HTTP 401: authentication_error: invalid api key
```

## Root Causes

1. **Incorrect Endpoint**: The configuration used the global endpoint `api.minimax.io`, but the user's API key is for the China-specific endpoint `api.minimaxi.com`. These are distinct environments with incompatible keys.

2. **Missing Auth Profile**: The `auth-profiles.json` in the `minimax-cn` agent directory did not contain an entry for the `minimax-cn` provider. It only had `minimax`.

3. **Conflicting Configuration in `models.json`**: The agent-level `models.json` contained hardcoded API keys. Crucially, it used the header `x-api-key` for authentication, whereas the China endpoint (`api.minimaxi.com`) specifically requires the standard `Authorization: Bearer` header format. This mismatch caused all API calls to fail with 401.

## Solution

1. Updated `openclaw.json` to point `minimax-cn` provider to `https://api.minimaxi.com/anthropic`.
2. Added `minimax-cn:default` profile to `auth-profiles.json` in both the main and `minimax-cn` agent directories.
3. Cleaned up `models.json` to remove hardcoded keys and redundant provider definitions, deferring to the global auth system and ensuring the correct authentication method is used.

## Verification

Verified that the key works with the endpoint `api.minimaxi.com/anthropic` using `Authorization: Bearer <key>`.

## Suggested Improvement

- Add better error handling to distinguish between "Invalid Key" and "Wrong Endpoint/Region".
- Consider adding a specific check for the `minimax-cn` provider configuration to warn users if they are using the wrong base URL.
