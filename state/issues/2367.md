## Description

## Summary
`clawdbot node install` generates a LaunchAgent plist that uses `node start` as the subcommand, but the CLI only supports `node run`. This causes the service to fail immediately on launch with:

```
error: unknown command 'start'
(Did you mean restart?)
```

## Steps to reproduce
1. `clawdbot node install`
2. `clawdbot node restart`
3. Check logs: `tail ~/.clawdbot/logs/node.err.log`

## Expected behavior
The plist should use `node run` as the subcommand.

## Actual behavior
The plist contains:
```xml
<string>node</string>
<string>start</string>
```

Should be:
```xml
<string>node</string>
<string>run</string>
```

## Workaround
Manually edit the plist:
```bash
clawdbot node stop
sed -i '' 's/<string>start<\/string>/<string>run<\/string>/' ~/Library/LaunchAgents/com.clawdbot.node.plist
launchctl bootout gui/$(id -u)/com.clawdbot.node
launchctl bootstrap gui/$(id -u) ~/Library/LaunchAgents/com.clawdbot.node.plist
```

## Environment
- Clawdbot version: 2026.1.21-2
- OS: macOS 15.7.3 (arm64)
- Install method: pnpm
