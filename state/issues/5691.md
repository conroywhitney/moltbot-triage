## Description

## Description

On openSUSE, Node.js is installed with versioned binary names like `/usr/bin/node24` instead of the standard `/usr/bin/node`. This causes two failures:

1. **Service generation** - `isNodeRuntime()` doesn't recognize `node24` as a valid Node binary
2. **CLI argument parsing** - Commander treats `/usr/bin/node24` as an unknown command

## Environment

- **OS:** openSUSE Tumbleweed (Linux 6.18.7-1-default)
- **Node:** v24.13.0 installed as `/usr/bin/node24`
- **OpenClaw:** 2026.1.30

## Node binary layout on openSUSE

```
$ ls -la /usr/bin/node*
-rwxr-xr-x. 1 root root    14520 Nov 28 13:43 /usr/bin/node      # libalternatives wrapper (14KB)
-rwxr-xr-x. 1 root root 56824888 Jan 22 06:53 /usr/bin/node24    # actual Node binary (56MB)

$ file /usr/bin/node /usr/bin/node24
/usr/bin/node:   ELF 64-bit LSB pie executable (libalternatives wrapper)
/usr/bin/node24: ELF 64-bit LSB pie executable (actual Node.js)
```

The `/usr/bin/node` wrapper uses `libalternatives.so.1` and sets `argv[0]` to `/usr/bin/node24`.

## Steps to reproduce

1. Install OpenClaw on openSUSE with `node24`
2. Run `openclaw gateway install`
3. Start the service: `systemctl --user start openclaw-gateway`

## Expected behavior

Gateway service starts successfully.

## Actual behavior

Service fails with:
```
error: unknown command '/usr/bin/node24'
```

## Root cause

The regex pattern `/^node-\d+/` requires a hyphen, so it matches `node-22` but not `node24`.

## Workaround

Manual sed to use the wrapper script instead of node directly:
```bash
sed -i 's|^ExecStart="/usr/bin/node"|ExecStart=/path/to/openclaw|' ~/.config/systemd/user/openclaw-gateway.service
```

## Fix

PR #5688 - Change regex to `/^node-?\d+/` (make hyphen optional)
