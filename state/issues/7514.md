## Description

**Bug: `sessions_spawn` model override not applied at inference time**

**Version:** 2026.2.1 (stable)

**Issue:** When spawning subagents with a model override (either via `sessions_spawn({model: "..."})` or `agents.defaults.subagents.model` config), the model is stored in the session but ignored at inference time. All requests go to the primary model (Opus).

**Evidence:**
- `sessions_spawn` returns `modelApplied: true`
- `session_status` shows correct model override (e.g., `xiaomi/mimo-v2-flash`)
- But `sessions_history` shows actual API calls used `anthropic/claude-opus-4.5`
- OpenRouter activity dashboard confirms only Opus was called

**Config:**
```json
{
  "models": {
    "providers": {
      "xiaomi": {
        "baseUrl": "https://openrouter.ai/api/v1",
        "api": "openai-completions",
        "models": [{"id": "mimo-v2-flash", ...}]
      }
    }
  },
  "agents": {
    "defaults": {
      "models": {
        "xiaomi/mimo-v2-flash": {"alias": "MiMo (Coding)"}
      },
      "subagents": {
        "model": "xiaomi/mimo-v2-flash"
      }
    }
  }
}
```

**Expected:** Subagent runs should use the configured/overridden model.
