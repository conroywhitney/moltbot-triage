hi @ajmeese7 , as far as I understand the update script, the failing of preflight linting is an expected behavior. It basically means that "this commit isn't a good one to update to your environment so it is skipped." 

What is valuable to you is:
_Update Result: ERROR
  Root: /home/ajmeese7/openclaw
  Reason: preflight-no-good-commit
  Before: 2026.1.29_

It's already told you that there's no good commit to udpate from the server. Your version is the most recent and stable one.

Hope that my explanation answers your question.

--update--
try the followings either:
```bash
openclaw update --channel stable
```
or
```bash
openclaw update wizard
```
and switch to the stable channel. I believe that the git install method set the 'dev' channel by default which causes the probelm.
