## Description

## Description
The `tts` tool does not use the configured ElevenLabs provider/voice, even when `messages.tts.provider` is set to `elevenlabs` with a valid `voiceId`.

## Steps to Reproduce
1. Configure TTS to use ElevenLabs:
```json
"messages": {
  "tts": {
    "auto": "inbound",
    "provider": "elevenlabs",
    "elevenlabs": {
      "voiceId": "1NHdu0ZEZCDyulQT37OL"
    }
  }
}
```
2. Set `ELEVENLABS_API_KEY` in `~/.openclaw/.env`
3. Use the `tts` tool to generate speech
4. The generated audio uses OpenAI voice (American accent) instead of the configured ElevenLabs voice (British accent)

## Expected Behavior
TTS should use ElevenLabs with the specified voice ID.

## Actual Behavior
TTS appears to fall back to OpenAI regardless of config.

## Workaround
Direct `curl` calls to the ElevenLabs API work correctly:
```bash
curl -s "https://api.elevenlabs.io/v1/text-to-speech/$VOICE_ID" \
  -H "xi-api-key: $ELEVENLABS_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{"text":"...","model_id":"eleven_multilingual_v2"}' \
  -o output.mp3
```

## Environment
- OpenClaw version: 2026.1.30
- OS: macOS Darwin 24.6.0
- ElevenLabs subscription: Starter
