## Description

The ⁠ openclaw message send ⁠ command fails when attempting to send media-only messages (without a text message) on WhatsApp, despite documentation suggesting this should be supported.

Expected Behavior:
According to ⁠ /usr/lib/node_modules/openclaw/docs/cli/message.md ⁠:
⁠For the ⁠ send ⁠ action: Required: ⁠ --target ⁠, plus ⁠ --message ⁠ or ⁠ --media ⁠

This implies either ⁠ --message ⁠ OR ⁠ --media ⁠ should work independently.

Additionally, ⁠ /usr/lib/node_modules/openclaw/docs/channels/whatsapp.md ⁠ states:
"Caption only on first media item" — suggesting captions are optional

Actual Behavior:
Running a media-only send fails with a validation error:
⁠ bash
openclaw message send --channel whatsapp --target +1234567890 --media /path/to/file.mp4
 ⁠

Error: ⁠ invalid send params: at /message: must NOT have fewer than 1 characters ⁠

Workaround:
Currently using a minimal character (e.g., ⁠ "." ⁠) as the message parameter.

Environment:
•⁠  ⁠Node version: 22.22.0
•⁠  ⁠OS: Linux
•⁠  ⁠Channel: WhatsApp (Baileys)

Reproduction:
1.⁠ ⁠Attempt to send media without message text via CLI or message tool
2.⁠ ⁠Observe validation error

Request:
Either fix the validation to allow media-only sends (matching documentation), or update documentation to reflect the requirement for message text.
