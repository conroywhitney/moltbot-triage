## Description

## Bug Description

The `model` parameter in `sessions_spawn` is completely ignored. All spawned subagents use the main session's model regardless of:

1. Config having `agents.defaults.subagents.model` set
2. Passing explicit `model` parameter to `sessions_spawn`
3. Response showing `modelApplied: true`

## Environment

- OpenClaw version: 2026.2.1
- OS: macOS (Darwin 25.2.0 arm64)
- Node: v25.5.0

## Configuration

```json
"agents": {
  "defaults": {
    "subagents": {
      "maxConcurrent": 8,
      "model": "gemini/gemini-2.5-flash-preview-05-20"
    }
  }
}
```

## Steps to Reproduce

1. Configure subagent model in `openclaw.json` (as shown above)
2. Verify Gemini API key is set (`GEMINI_API_KEY` env var)
3. Spawn a subagent with explicit model parameter:

```typescript
sessions_spawn({
  label: 'test-gemini',
  task: 'Report what model you are running on',
  model: 'gemini/gemini-2.5-flash-preview-05-20'
})
```

4. Response shows `modelApplied: true`
5. Check subagent session history - shows `provider: anthropic, model: claude-opus-4-5`

## Expected Behavior

Subagent should use the specified model (`gemini/gemini-2.5-flash-preview-05-20`)

## Actual Behavior

Subagent uses main session's model (`claude-opus-4-5`) despite `modelApplied: true` response

## Impact

- Unable to use cheaper models for subagents
- Significantly increased API costs
- Cost discipline impossible for multi-agent workflows

## Workaround

None known.
